# prdash UX polish Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Land the six UX-polish items from issue #3 on the prdash TUI.

**Architecture:** Bottom-up — data/struct changes first, then pure renderers, then
keybindings, then the responsive expanded layout, finally perf/cache and a live
audit. TDD where the unit is a pure function; live-driven for the bug + audit.

**Tech Stack:** Go, charm.land bubbletea/v2 + lipgloss/v2 + glamour, `gh` CLI.

## Global Constraints

- Cache `schemaVer` bumps `v2` → `v3` (new `color` + `detailsUrl` fields).
- Rerun keys: `r` = hovered check, `R` = all failed (not `ctrl+r`/`ctrl+R`).
- Commit with `PRE_COMMIT_ALLOW_NO_CONFIG=1` (nix symlink absent in worktree).
- `go test ./...` and `go build ./...` green before each commit.
